# tw-stock-screen skill

把台灣AI供應鏈選股系統化：用四條件框架評估個股、找遺漏標的、或生成可執行的篩選腳本。

## 何時用

| 情境 | 用 | 不用 |
|---|---|---|
| 分析某支股票是否符合框架 | ✅ | |
| 找更多符合條件的標的 | ✅ | |
| 寫選股腳本 | ✅ | |
| 查股票即時報價 | | ❌ 直接查即時行情 |
| 預測短期漲跌 | | ❌ 框架是長線結構性分析 |
| 判斷買賣時機 | | ❌ 框架不做擇時 |

## 四條件框架

詳細評分標準與案例見 `references/four-criteria.md`。

| 條件 | 可量化 | 主要信號來源 |
|---|---|---|
| 供應鏈關鍵節點 | ⚠️ 定性為主 | 市占率、競爭者數量、Hyperscaler點名 |
| 結構性需求 | ✅ 可量化 | 多季YoY成長、AI佔比趨勢、客戶結構 |
| 高替換成本 | ❌ 純定性 | 認證週期、共同開發深度、技術稀缺性 |
| 法人長期持有 | ✅ 可量化 | 外資持股%、連續買超週數、目標價動向 |

## 工具選擇

搜尋資料時依序嘗試：

1. `mcp__linkup__linkup-search`（主力，覆蓋財經媒體）
2. `mcp__jina__search_web`（備用）
3. `mcp__firecrawl__firecrawl_search`（需確認額度）

每次分析一支股票，至少搜尋以下三個角度：
- `公司名 + AI供應鏈 + 市占率 + 競爭格局`
- `公司名 + 外資持股 + 法說會 + 2025 2026`
- `公司名 + 護城河 + 替換成本 OR 認證`

## 執行模式

### 模式 A：單股分析

1. 搜尋上述三個角度的資料
2. 對四個條件逐一評分（參照 `references/four-criteria.md` 的評分標準）
3. 輸出標準評分卡（格式見文末）

### 模式 B：擴充名單

使用者要找「更多符合條件的標的」時：

1. 確認目前名單涵蓋的供應鏈層級
2. 系統性找遺漏環節，依以下層級搜尋：
   - 上游材料（CCL、矽晶圓、化學品）
   - 基板與載板（ABF、FC-BGA）
   - 晶片設計（ASIC、IP授權）
   - 製造與封裝（晶圓代工、先進封裝、測試）
   - 散熱與電源
   - 光通訊（CPO、磊晶）
   - 網通設備
3. 每個候選跑模式 A，只列入確認符合全四條件的

### 模式 C：生成腳本

輸出量化 + LLM 混合的 Python 篩選腳本。

架構固定為三層：
- Layer 1：量化篩選（FinMind API，自動淘汰 90%）
- Layer 2：LLM 定性評分（Claude API，評分條件①③）
- Layer 3：人工複核（最終 20-30 檔候選）

完整程式碼骨架見 `references/screening-code.md`。

## 反合理化

| 想偷懶 | 為什麼不行 |
|---|---|
| 只看財務數字、跳過定性條件 | 條件①③才是護城河來源，跳過等於沒篩 |
| 沒找到明確 evidence 還是給高分 | 不確定就給低分，這個框架不做假設 |
| 把「大家都看好」當法人持有依據 | 要看實際外資持股%和買超趨勢，不是情緒 |
| 把週期性需求當結構性需求 | 消費電子旺季 ≠ 結構性；AI Capex 才算 |
| 腳本跑完就不人工驗證 | LLM 會幻覺，evidence 驗證步驟不能省 |
| 把「供應商之一」當關鍵節點 | 暗示有多家可選，應評 2-3 分而非 5 分 |
