## System Prompt 三段式結構

### 1. 角色定義（Role）
```
你是一個 [具體角色]，專精於 [領域]。你的目標是 [具體目標]。
```
具體 > 抽象：「你是資深 RAG 工程師」比「你是 AI 助手」好

### 2. 行為準則（Guidelines）
```
## 回答規則
- 不確定時說「我不確定」，不要猜
- 先給結論，再解釋原因
- 程式碼要可執行，不要虛擬碼
```
正面描述優於負面描述，但關鍵禁止事項要明確列出

### 3. 輸出格式（Format）
```
## 輸出格式
回答請包含：
1. 一句話結論
2. 技術細節（含程式碼）
3. 注意事項
```

## Context 格式化原則

**語意清晰優於原始拼接**：
```
## 相關文件
### [文件標題]（來源：xxx，相關度：高）
[內容摘要]
```

**Token 預算**：預留 30% context window 給生成，最相關文件放前面（primacy effect）

**信心機制**：在 system prompt 加入「根據目前的資料無法確定」的明確引導

## 推理框架速查

| 框架 | 適用場景 | 觸發語 |
|------|---------|--------|
| **CoT** | 數學推理、邏輯問題、多步驟分析 | 「請一步一步思考」 |
| **Few-shot** | 格式轉換、分類、風格模仿 | 提供 2-3 個 input → output 範例 |
| **ReAct** | 需要多步驟檢索的問題 | Thought → Action → Observation 迴圈 |

## Prompt 迭代方法

1. 從最簡單的 prompt 開始
2. 用真實 case 測試，不要用假設的例子
3. 發現錯誤 → 分析類型（理解？格式？知識？）
4. 針對性修改，不要大改
5. 記錄每次改動和結果（prompt 也需要版本控制）

## 常見錯誤

- 塞太多規則 → LLM 忽略後面的
- 用否定句 → 改用肯定句（「當遇到 X 時，做 Y」）
- 沒給範例 → 格式不穩定
- prompt 太長 → 重點被稀釋，拆成多輪
