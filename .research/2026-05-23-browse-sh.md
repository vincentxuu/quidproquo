## 草稿骨架
（以下段落可直接丟給 `post` skill 的 `tech-deep-dive` 模板，category 建議 `ai`）

### 核心概念
browse.sh 是 Browserbase 在 2026-05 推出的東西，本質是**兩件事綁一起**：(1) 一個開放的「瀏覽器技能(browser skills)」目錄，發布時 100+ 個精選技能；(2) Browse CLI(`npm i -g browse`)，讓 AI Agent 真正去開瀏覽器、抓頁面、搜尋網路、按需載入技能。它在解的問題不是「特色清單」，而是一句話：**瀏覽器 Agent 的瓶頸從來不是智商，是健忘症**——每次跑都從零重新摸索同一個網站(開瀏覽器、亂點、找 selector、解析、關閉、忘光)，這個重複成本他們叫 discovery tax(探索稅)。skill 就是把「上次學到的」存成耐用、可重用、人能讀 Agent 能跑的 playbook。

⚠️ **消歧義(務必在文章開頭講)**：browse.sh 跟 2018 年那個叫 **Browsh / brow.sh** 的終端機文字瀏覽器(Thomas Buckley-Houston 作、用 headless Firefox 把網頁渲染成 TTY 文字、LGPL、最後版本 1.8.3 在 2024-01)**完全是兩個東西**，只是名字像。

### 關鍵設計決定
- **技能是純文字 markdown(SKILL.md)+ 腳本**，刻意不用向量嵌入、不用截圖串流——理由是要「人能 audit、Agent 能執行」。技能裡寫的是確切步驟、踩雷點(gotchas)、隱藏 API endpoint、selector、fallback 策略。
- **記憶優先於推理**：官方明說「就算模型完美，每到一個新站還是得重新發現一個完美模型若來過就會知道的事」,所以要有個「放學到的東西」的地方。
- **技能用 Autobrowse「養」出來**：給 Agent 一個真實任務跑到底 → 讀自己的 trace → 把觀察寫進 `strategy.md` 當下一輪 context → 迭代(上限約 3–5 次)→ 成本/回合數不再明顯下降就收斂 → 寫出 SKILL.md。第一次跑故意貴,它付的是「之後每次都便宜」的錢。
- **接 AgentSkills 開放標準**:SKILL.md 格式源自 Anthropic 開源的標準(progressive disclosure:啟動只載 name+description,命中才讀全文),Claude Code、Codex 都吃這套。browser skills 是這浪潮的自然延伸。
- **read-only 為主**:目錄裡多數技能標 read-only(不下單、不送出),敏感任務(IRS 退稅、USCIS 案件)強調不記 PII。

### 跟替代方案的比較
- **vs 手刻 Playwright 腳本**:Browserbase 的賣點是「Agent 已經在上週解過的工作流,你還在手寫 Playwright」。技能是 Agent 自己養的、會隨網站改版重新 graduate。
- **vs 通用 browser agent loop(裸跑)**:同樣任務,Craigslist 從 ~$0.22/71s 降到 ~$0.12/27s(官方 blog 稱約 45%);首頁另宣稱建議 selector + XHR「省 50x token」(⚠️行銷語、單源)。
- **vs 純 deterministic 解析**:Autobrowse 官方自己承認**不該硬上**——對 167 列靜態 HTML 硬跑 4 輪燒了 ~$24 還沒抓全,改 200 行 BeautifulSoup 秒殺。「能 fetch 就別開瀏覽器,能 deterministic 就別上 Autobrowse」。

### 適合 / 不適合的情境
- **適合**:在 production 跑 browser agent、被重複 discovery 成本壓著的人;要把網站行為變成可 audit/可交接 playbook 的團隊;用 Claude Code / Cursor / Codex 想讓 coding agent「帶著現成知識」上網的人。
- **不適合**:任務是 deterministic parsing(資料就在 markup 裡)→直接寫 parser;只查單一事實→直接 fetch/search;不想碰雲端 browser/付費 API 的純本機輕量需求。

### 限制 / 已知問題
- **「開源免費」要打星號**:CLI 與技能開源(skills repo MIT),但實際跑完整工作流可能要模型額度、Browserbase 憑證、雲端 session、residential proxy、解 captcha、付費 API——open source ≠ 全程零成本。
- **品牌/命名還在收斂**:同時看得到 `browse`(browse.sh 的獨立 CLI)、`bb`(Browserbase CLI,`bb browse` 是 passthrough)、`@browserbasehq/browse-cli`(npm)、`browse.sh` 與 `skills.sh/browserbase` 兩個目錄域名並存。寫文章時要點出這層混亂、別當成單一乾淨產品。
- **技能可靠度依賴網站不變**:網站 Tuesday 改版,技能就可能要重新 graduate;Autobrowse 收斂是「夠用」不是全域最優。
- **數據多為官方自評**:成本數字、50x 都來自 Browserbase 自己,缺第三方獨立 benchmark。

### 取捨總結
用「把 Agent 學到的東西寫成純文字技能」換「不再付重複探索稅」——賭的是 browser agent 的未來瓶頸是記憶不是推理。代價是:依賴官方 Autobrowse/平台生態、數據自評、品牌命名仍亂、且只在「需要探索」的網站上划算(deterministic 的活硬上反而更貴)。一句話總結官方的賣點:**瓶頸不是智商,是健忘症;browse.sh 是解藥。**
